# 康乃馨采摘机器人视觉识别项目简历素材

## 项目名称

基于 YOLOv8 与 ROS2 的康乃馨自动采收机视觉识别与目标定位系统

## 项目目标

面向康乃馨自动采收机器人，构建“图像采集 -> YOLOv8 目标检测 -> 像素中心提取 -> 坐标转换 -> ROS2 发布目标位姿”的视觉感知链路，为后续机械臂采摘规划提供目标位置输入。项目定位为数字样机与软件仿真验证，不做真机部署。

## 项目职责

- 搭建康乃馨目标检测数据集，完成训练集、验证集目录组织与标签管理。
- 使用 LabelImg 完成康乃馨花头单类别标注，类别为 `carnation_flower`。
- 将 Pascal VOC `.xml` 标注批量转换为 YOLO `.txt` 格式，保证 YOLOv8 可直接训练。
- 在 Windows 侧配置 YOLOv8 训练环境，基于 Python 3.12、Ultralytics 和 PyTorch CUDA 完成模型训练。
- 分别训练并对比 YOLOv8n、YOLOv8s、YOLOv8m 模型，分析 Precision、Recall、mAP、F1、混淆矩阵和训练曲线。
- 编写批处理脚本，支持一键训练、续训、测试集推理、单张图片推理、摄像头推理和屏幕区域实时识别。
- 规划 ROS2 视觉模块后续接入方案，包括 `carnation_perception` 检测节点和 `carnation_localization` 坐标转换节点。
- 明确后续 ROS2 链路为：检测框中心点 `(u, v)` -> 相机坐标/世界坐标 -> 发布 `/target_pose`。

## 技术栈

- 操作系统：Windows，WSL Ubuntu 24.04
- 机器人系统：ROS2 Jazzy
- 深度学习：YOLOv8，Ultralytics，PyTorch CUDA
- 视觉处理：OpenCV，Pillow
- 标注工具：LabelImg 1.8.6
- 数据格式：Pascal VOC，YOLO txt
- 编程语言：Python，Batch 脚本
- 工程方向：机器人视觉感知，目标检测，坐标转换，ROS2 topic 通信，数字样机验证

## 解决的问题

- 解决康乃馨目标形态差异大的问题，将盛开、半盛开、不同颜色样本统一为单类别目标检测任务。
- 解决 LabelImg 在新版 Python/PyQt 环境下的闪退和绘图兼容问题，保证标注流程可用。
- 解决 Pascal VOC 标注无法直接用于 YOLOv8 训练的问题，完成批量格式转换。
- 解决 Windows 训练过程中页面文件不足、训练脚本解析异常、续训参数不生效等问题。
- 通过对比不同 YOLOv8 模型，分析模型容量、训练轮数与过拟合之间的关系。
- 为后续 ROS2 集成明确从视觉检测结果到 `/target_pose` 的工程接口。

## 成果亮点

- 完成康乃馨花头单类别检测模型训练，构建可用于后续 ROS2 感知节点集成的 `best.pt` 模型。
- 建立从数据标注、格式转换、模型训练、模型对比到推理验证的完整目标检测流程。
- 训练集中目标实例数量约 3600 个，覆盖多目标、不同距离、遮挡、角度、光照和颜色变化。
- YOLOv8n 初版模型达到 `mAP@0.5 = 0.851`，具备较好的基础检测能力。
- YOLOv8s 相比 YOLOv8n 减少误检和漏检，混淆矩阵中正确检出由 `321` 提升至 `337`，误检由 `105` 降至 `88`，漏检由 `74` 降至 `58`。
- 通过训练曲线分析发现 YOLOv8m 在 150 轮训练下出现过拟合趋势，并将训练轮数优化至 80 轮。
- 编写多个自动化脚本，降低训练、推理和验证操作成本。

## 可量化指标

- 数据集类别数：1 类，`carnation_flower`
- 训练集图片数量：约 301 张
- 验证集图片数量：40 张
- 标注目标实例数：约 3600 个
- YOLOv8n：`mAP@0.5 = 0.851`
- YOLOv8n：F1 最优约 `0.80`，对应置信度约 `0.475`
- YOLOv8s：正确检出 `337`，误检 `88`，漏检 `58`
- YOLOv8s 相比 YOLOv8n：正确检出增加 `16`，误检减少 `17`，漏检减少 `16`
- 训练硬件：NVIDIA RTX 3060 Laptop GPU
- 训练模型：YOLOv8n，YOLOv8s，YOLOv8m
